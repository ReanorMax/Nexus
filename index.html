Установка и настройка Nexus Sonatype

Nexus Sonatype — это менеджер репозиториев, который обеспечивает централизованное хранение и управление вашими артефактами. Он значительно упрощает процесс разработки программного обеспечения и обеспечивает безопасность ваших компонентов.

Централизованное хранение

Храните и управляйте всеми вашими артефактами в одном месте с возможностью контроля версий

Гибкая настройка

Настраивайте доступ к репозиториям и управляйте правами для различных команд разработчиков

Поддержка различных форматов

Работайте с Maven, npm, Docker, APT, RPM и другими форматами в одной системе

Зачем нужен Sonatype Nexus?

  • Хранение и управление артефактами
  • Проксирование внешних репозиториев для кеширования артефактов
  • Управление зависимостями в проектах
  • Обеспечение безопасности и контроля доступа к артефактам

Что такое артефакты в Nexus?

В контексте Nexus артефакт — это любой бинарный файл, который хранится в репозитории. Артефакты могут быть разных типов и имеют определенную структуру.

Типы артефактов

Библиотеки

JAR-файлы для Java, DLL для .NET

Образы

Docker-образы и контейнеры

Установочные файлы

RPM, DEB пакеты

Другие файлы

ZIP-архивы, конфигурации

Структура артефактов

Имя

Например, my-library

Версия

Например, 1.0.0

Тип

Например, JAR, WAR, TGZ

Типы репозиториев в Nexus

Nexus поддерживает множество типов репозиториев, и не все они связаны с артефактами.

Raw

Для произвольных файлов (например, ZIP, TXT, JSON)

Пример: raw-hosted

RPM

Для пакетов Linux (RPM)

Пример: rpm-hosted

Binary

Для бинарных файлов (например, .zip, .tar.gz, .exe)

Пример: binary-repo

Установка Nexus с Docker

Пошаговое руководство по настройке Docker и развертыванию Nexus Repository на вашем сервере

Установка Docker на Debian

Перед установкой Nexus Sonatype, необходимо настроить Docker на вашем сервере.

1

Обновление системы

Убедитесь, что ваша система обновлена до последней версии:

sudo apt update
sudo apt upgrade -y
2

Установка необходимых пакетов

Установите пакеты, которые позволят использовать репозиторий через HTTPS:

sudo apt install -y apt-transport-https ca-certificates curl software-properties-common gnupg lsb-release
3

Добавление GPG-ключа Docker

Добавьте официальный GPG-ключ Docker для проверки подлинности репозитория:

curl -fsSL https://download.docker.com/linux/debian/gpg | sudo gpg --dearmor -o /usr/share/keyrings/docker-archive-keyring.gpg
4

Добавление репозитория Docker

Добавьте репозиторий Docker в список источников APT:

echo "deb [arch=$(dpkg --print-architecture) signed-by=/usr/share/keyrings/docker-archive-keyring.gpg] https://download.docker.com/linux/debian $(lsb_release -cs) stable" | sudo tee /etc/apt/sources.list.d/docker.list > /dev/null
5

Обновление индекса пакетов

Обновите индекс пакетов после добавления репозитория:

sudo apt update
6

Установка Docker

Установите Docker Engine, Docker CLI и containerd:

sudo apt install -y docker-ce docker-ce-cli containerd.io docker-buildx-plugin docker-compose-plugin
7

Запуск и включение Docker

Запустите Docker и настройте его автозапуск при загрузке системы:

sudo systemctl start docker
sudo systemctl enable docker
8

Проверка установки Docker

Убедитесь, что Docker установлен и работает:

docker --version
sudo systemctl status docker

Установка Nexus Repository Manager 3 в Docker

После успешной установки Docker, вы можете запустить Nexus в контейнере

1

Создание директории для хранения данных Nexus

Создайте директорию на хосте для хранения данных Nexus, чтобы они сохранялись при перезапуске контейнера:

sudo mkdir -p /opt/nexus-data sudo chown -R 200 /opt/nexus-data

UID 200 соответствует пользователю внутри контейнера Nexus, чтобы избежать проблем с правами доступа.

2

Запуск контейнера Nexus

Запустите контейнер Nexus с использованием официального образа sonatype/nexus3:

docker run -d \ --name nexus \ -p 8081:8081 \ -v /opt/nexus-data:/nexus-data \ sonatype/nexus3:latest

Объяснение параметров:

  • -d — запуск в фоновом режиме.
  • --name nexus — имя контейнера.
  • -p 8081:8081 — проброс порта 8081 для доступа к веб-интерфейсу.
  • -v /opt/nexus-data:/nexus-data — монтирование директории для хранения данных.
  • sonatype/nexus3:latest — использование последней версии образа Nexus.
3

Проверка статуса контейнера

Убедитесь, что контейнер запущен:

docker ps

Если контейнер не работает, проверьте логи для диагностики:

docker logs nexus
4

Ожидание запуска Nexus

Первый запуск Nexus может занять несколько минут, так как он создаёт необходимые файлы конфигурации и базы данных. Следите за логами, чтобы убедиться, что Nexus запустился:

docker logs -f nexus

Когда в логах появится сообщение вроде Started Sonatype Nexus OSS, Nexus готов к использованию.

5

Доступ к веб-интерфейсу

Откройте браузер и перейдите по адресу:

http://<IP-адрес-сервера>:8081

Например, если сервер имеет IP 10.19.1.216, используйте:

http://10.19.1.216:8081
6

Первый вход в Nexus

Логин: Используйте имя пользователя admin.

Пароль: Пароль администратора сгенерирован автоматически и находится в файле /opt/nexus-data/admin.password. Посмотрите его с помощью:

sudo cat /opt/nexus-data/admin.password

После входа система попросит вас сменить пароль. Установите новый безопасный пароль.

Создание APT Репозиториев

Настройка и использование APT репозиториев в Nexus для управления Debian-пакетами

Создание APT Hosted Репозитория

Локальный репозиторий для хранения ваших собственных .deb пакетов

1

Создание нового репозитория

Перейдите в раздел Repositories → нажмите Create repository.

Выберите формат apt (hosted).

2

Настройка репозитория

Заполните следующие поля:

  • Name: local-deb-repo (имя вашего локального репозитория).
  • Format: apt.
  • Layout: default.
  • Distribution: bookworm (ваш дистрибутив Debian).
  • Sections: main (или добавьте дополнительные секции, если нужно: contrib, non-free).
  • Deployment Policy: Выберите Disable Redeploy (рекомендуется для production) или Allow Redeploy (для тестирования).

Нажмите кнопку Create repository.

3

Загрузка .deb пакетов

Перейдите в раздел Repositories → выберите local-deb-repo.

Нажмите кнопку Upload.

Выберите все ваши .deb файлы (например, Asterisk, Zabbix, PHP и т.д.) и загрузите их.

Примечание: Nexus автоматически определит правильную структуру каталогов (pool/main/...) для каждого пакета.

4

Обновление метаданных

После загрузки пакетов обновите метаданные репозитория:

Через веб-интерфейс: Нажмите кнопку Rebuild Metadata.

Создание APT Proxy Репозитория

Proxy репозиторий для кэширования пакетов из официальных источников Debian/Ubuntu

1

Создание Proxy репозитория

Перейдите в раздел Repositories → нажмите Create repository.

Выберите формат apt (proxy).

2

Настройка Proxy репозитория

Заполните следующие поля:

  • Name: debian-bookworm-proxy (имя proxy репозитория).
  • Format: apt.
  • Layout: default.
  • Remote storage URL: Укажите URL официального репозитория Debian Bookworm
http://deb.debian.org/debian
  • Distribution: bookworm.
  • Sections: main contrib non-free.

Нажмите кнопку Create repository.

Настройка доступа и использование репозиториев

Настройка анонимного доступа и конфигурация клиентских систем

1

Настройка доступа

Если вы хотите разрешить анонимный доступ:

  • Перейдите в раздел Security → Anonymous.
  • Включите опцию Allow anonymous access.
2

Важная информация о Group репозиториях

В бесплатной версии Nexus Repository Manager поддержка Group репозиториев отсутствует. Вместо этого вы можете настроить два отдельных репозитория:

  • APT Hosted для ваших локальных пакетов
  • APT Proxy для кэширования официальных репозиториев Debian/Ubuntu

Затем клиентские системы будут использовать оба репозитория одновременно.

3

Настройка клиентской системы

Добавьте оба репозитория в sources.list:

Создайте или отредактируйте файл /etc/apt/sources.list.d/nexus.list:

# Локальный репозиторий с вашими пакетами deb [signed-by=/usr/share/keyrings/nexus-key.gpg] http://10.19.1.216:8081/repository/local-deb-repo/ bookworm main # Proxy репозиторий для зависимостей deb [signed-by=/usr/share/keyrings/debian-archive-keyring.gpg] http://10.19.1.216:8081/repository/debian-bookworm-proxy/ bookworm main contrib non-free
4

Как это работает?

APT Hosted репозиторий (local-deb-repo):

  • Хранит ваши локальные пакеты.
  • Не требует подключения к интернету после загрузки пакетов.

APT Proxy репозиторий (debian-bookworm-proxy):

  • Кэширует официальные репозитории Debian Bookworm.
  • Предоставляет зависимости для ваших пакетов.

Клиентская система:

  • Использует оба репозитория одновременно.
  • APT автоматически разрешает зависимости, проверяя сначала local-deb-repo, а затем debian-bookworm-proxy.
5

Процесс разрешения зависимостей

APT будет выполнять следующие действия при установке пакетов:

  • Искать Asterisk, Zabbix и другие ваши пакеты в local-deb-repo.
  • Искать зависимости (например, libc6, libssl3, libsqlite3-0) в debian-bookworm-proxy.

Преимущества такого подхода:

  • Простота настройки: Не требуется создавать Group репозиторий (который недоступен в бесплатной версии Nexus).
  • Автоматическое разрешение зависимостей: Proxy репозиторий обеспечивает доступ ко всем необходимым зависимостям из официальных источников.
  • Гибкость: Вы можете добавлять новые пакеты в local-deb-repo без необходимости изменять конфигурацию клиентских систем.

Это решение обеспечивает:

  • Локальное хранение ваших пакетов.
  • Автоматическое разрешение зависимостей через proxy репозиторий.
  • Минимальные изменения в конфигурации клиентских систем.
6

Пример использования и проверка

Обновление списка пакетов:

sudo apt update

Проверка доступности пакетов:

apt list -a asterisk apt list -a php8.2* apt list -a mysql-server apt list -a zabbix-server-mysql

Установка пакетов:

sudo apt install asterisk php8.2 php8.2-cli php8.2-common php8.2-mysql mysql-server zabbix-server-mysql

APT автоматически:

  • Возьмет ваши локальные пакеты из local-deb-repo.
  • Возьмет зависимости из debian-bookworm-proxy.

Создание .deb пакетов

Инструкция по созданию .deb пакета из бинарного файла или скомпилированного ПО с использованием checkinstall

Создание .deb пакета с использованием checkinstall

Процесс создания Debian-пакетов из исходного кода для последующего размещения в вашем APT репозитории

1

Установка необходимых инструментов

Убедитесь, что на вашей системе установлен checkinstall:

sudo apt update sudo apt install checkinstall
2

Сборка программы

a) Получите исходный код программы

Скачайте исходный код программы (например, Asterisk):

wget https://downloads.asterisk.org/pub/telephony/asterisk/asterisk-18.26.1.tar.gz tar -xzf asterisk-18.26.1.tar.gz cd asterisk-18.26.1

b) Установите зависимости для сборки

Проверьте документацию программы, чтобы узнать список зависимостей. Например, для Asterisk:

sudo apt install -y build-essential uuid-dev libssl-dev libedit-dev libjansson-dev libsqlite3-dev libpq-dev

c) Настройте и соберите программу

Выполните стандартные шаги для компиляции:

./configure --with-jansson-bundled --with-pjproject-bundled make
3

Создание .deb пакета с помощью checkinstall

После успешной сборки, запустите checkinstall для создания .deb пакета:

sudo checkinstall --install=no --pkgname=asterisk --pkgversion=18.26.1 --provides=asterisk --default

Объяснение параметров:

  • --install=no: Отключает автоматическую установку пакета в систему.
  • --pkgname=asterisk: Указывает имя пакета.
  • --pkgversion=18.26.1: Указывает версию пакета.
  • --provides=asterisk: Указывает, что этот пакет предоставляет функциональность Asterisk.
  • --default: Использует стандартные параметры для создания пакета.
4

Загрузка пакета в Nexus Repository

После создания .deb пакета, вы найдете его в текущей директории. Теперь вы можете загрузить его в ваш APT Hosted репозиторий в Nexus:

  1. Войдите в веб-интерфейс Nexus Repository Manager.
  2. Перейдите в раздел Repositories и выберите ваш local-deb-repo.
  3. Нажмите кнопку Upload и загрузите созданный .deb пакет.
  4. После загрузки не забудьте нажать Rebuild Metadata.
5

Автоматизация процесса

Для автоматизации процесса создания и загрузки пакетов вы можете использовать скрипты. Пример скрипта для автоматического создания .deb пакета:

#!/bin/bash # Пример автоматизации создания пакета PKG_NAME="asterisk" PKG_VERSION="18.26.1" SRC_URL="https://downloads.asterisk.org/pub/telephony/asterisk/asterisk-${PKG_VERSION}.tar.gz" # Создание временной директории WORK_DIR=$(mktemp -d) cd "$WORK_DIR" # Скачивание и распаковка исходников wget "$SRC_URL" tar -xzf "asterisk-${PKG_VERSION}.tar.gz" cd "asterisk-${PKG_VERSION}" # Установка зависимостей sudo apt install -y build-essential uuid-dev libssl-dev libedit-dev libjansson-dev libsqlite3-dev libpq-dev # Конфигурация и сборка ./configure --with-jansson-bundled --with-pjproject-bundled make # Создание .deb пакета sudo checkinstall --install=no --pkgname="$PKG_NAME" --pkgversion="$PKG_VERSION" --provides="$PKG_NAME" --default # Перемещение пакета в нужную директорию mv *.deb ~/packages/ echo "Создание пакета завершено. Пакет находится в ~/packages/"

Другие типы репозиториев

Настройка репозиториев для хранения и управления различными типами файлов

Создание репозитория для бинарных файлов

Хранение и управление произвольными бинарными файлами в Nexus Repository Manager

1

Создание нового репозитория

Перейдите в раздел Repositories → нажмите Create repository.

Выберите тип репозитория raw (hosted).

2

Настройка репозитория

Заполните следующие поля:

  • Name: Укажите имя репозитория (например, binary-repo).
  • Repository Format: Выберите Raw.
  • Storage: Убедитесь, что путь к хранилищу указан правильно.

Нажмите кнопку Create repository.

3

Загрузка файлов

Перейдите в созданный репозиторий (например, binary-repo).

Нажмите Upload.

Выберите файлы для загрузки (например, .zip, .tar.gz, .exe и т.д.).

Укажите путь для файла в репозитории (например, /path/to/file.zip).

Нажмите Upload.

4

Проверка загруженных файлов

После загрузки файлов в Nexus убедитесь, что они доступны:

  • Перейдите в веб-интерфейс Nexus → "Browse" → выберите репозиторий.
  • Убедитесь, что структура папок и файлов соответствует вашим ожиданиям.
5

Использование файлов

Для доступа к файлам используйте следующий формат URL:

http://<IP-адрес-сервера>:8081/repository/<имя-репозитория>/<путь-к-файлу>

Например:

http://10.19.1.216:8081/repository/binary-repo/path/to/file.zip

Пояснение к визуализации

Hosted репозиторий — хранит артефакты, созданные вашей организацией. Это полностью управляемое вами хранилище для ваших внутренних проектов и библиотек.

Proxy репозиторий — кэширует артефакты из внешних источников.

Group репозиторий — объединяет несколько репозиториев в единую точку доступа.

Эта визуализация демонстрирует ключевое преимущество использования Nexus: один раз загруженный внешний артефакт кэшируется локально, что ускоряет сборку проектов и снижает зависимость от внешних источников.
